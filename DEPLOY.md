# Деплой на Cloudflare (бесплатно, с динамикой)

Сайт работает на Cloudflare через адаптер **`@opennextjs/cloudflare`** — это
полноценный Next.js (живые цены из Google-таблицы, серверный рендер), а не
статика. Бесплатный тариф, приватный репозиторий — ок, свой домен + бесплатный SSL.

## 1. Подключить репозиторий

1. <https://dash.cloudflare.com> → **Workers & Pages** → **Create** →
   вкладка **Workers** → **Import a repository** → подключить GitHub и выбрать
   `shataev13/salon-site`, ветка **main**.
2. Настройки сборки:
   - **Build command:** `npx opennextjs-cloudflare build`
   - **Deploy command:** `npx wrangler deploy`
   - Остальное Cloudflare возьмёт из `wrangler.jsonc` (имя воркера, флаги
     `nodejs_compat` и т.п.).
3. **Deploy.** Через пару минут получите рабочий адрес `…workers.dev`.

Дальше каждый `git push` в `main` будет автоматически пересобирать и
выкатывать сайт.

## 2. Переменные окружения (необязательно)

Всё работает со значениями по умолчанию (онлайн-запись и ID таблицы зашиты).
При желании переопределить — в настройках воркера **Settings → Variables**:

| Переменная                | Зачем                                   |
| ------------------------- | --------------------------------------- |
| `NEXT_PUBLIC_BOOKING_URL` | Ссылка на онлайн-запись YClients        |
| `GOOGLE_SHEET_ID`         | ID Google-таблицы с прайсом             |
| `GOOGLE_SHEET_GID`        | gid листа                               |
| `GOOGLE_SHEET_REVALIDATE` | Период обновления прайса, сек (300)     |

> Чтобы цены тянулись из таблицы — таблица должна быть открыта на чтение
> («Поделиться → Доступ по ссылке → Читатель»).

## 3. Свой домен (куплен на reg.ru)

Самый простой путь с бесплатным SSL — перенести DNS домена в Cloudflare:

1. Cloudflare → **Websites** → **Add a site** → ваш домен → план **Free**.
2. Cloudflare покажет **2 своих NS-сервера** — пропишите их в reg.ru
   (управление доменом → DNS-серверы / NS) вместо текущих.
3. После активации домена: воркер → **Settings → Domains & Routes** →
   **Add → Custom domain** → ваш домен (и `www`). SSL выпустится сам.

Если переносить NS не хотите — можно оставить DNS на reg.ru и добавить
CNAME/записи, которые подскажет Cloudflare при добавлении custom domain.

## 4. Локальные команды

- `npm run cf:preview` — собрать и посмотреть локально (эмуляция Worker).
- `npm run cf:deploy` — собрать и выкатить вручную (нужен `npx wrangler login`).
- Обычная разработка — как раньше: `npm run dev`.

## Заметки

- **Картинки** отдаются без серверной оптимизации (на Cloudflare нет встроенного
  оптимизатора `next/image`) — поэтому они заранее сжаты в WebP/JPG.
- **Живые цены** из таблицы работают: запрос идёт из воркера при заходе на
  страницу услуги (обновление — по `GOOGLE_SHEET_REVALIDATE`).
